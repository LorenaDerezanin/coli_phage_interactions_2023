Create a pull request using `gh pr create`. Follow the `/gh` skill guardrails — especially the HEREDOC body requirement.

## Orchestrator task PR

If this PR implements an orchestrator task (the user mentioned a task ID or issue number):

```bash
gh pr create \
  --title "[ORCH][TASK_ID] Brief description" \
  --label orchestrator-task \
  --body "$(cat <<'EOF'
## Summary
<1-3 bullet points>

## Test plan
- [ ] Tests pass
- [ ] Acceptance criteria met

Closes #<issue_number>

🤖 Generated by <agent model name>
EOF
)"
```

## Regular PR

```bash
gh pr create \
  --title "Brief description" \
  --body "$(cat <<'EOF'
## Summary
<1-3 bullet points>

## Test plan
- [ ] Tests pass

🤖 Generated by <agent model name>
EOF
)"
```

## Key rules (from /gh skill)

1. **HEREDOC is mandatory** — never use `--body "text\n\nCloses #N"`. The `\n` produces literal backslash-n, not newlines.
2. **`Closes #N` on its own line** — preceded by a blank line, at the start of the line.
3. **Agent transparency** — identify yourself if posting under a human's credentials.
4. Use `<<'EOF'` (single-quoted) to prevent shell expansion. Use `<<EOF` only when you need `$variable` interpolation.
